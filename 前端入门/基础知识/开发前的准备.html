# 文件命名规范和标签的概念

# 文件名

​ 文件名可以使用数字、字母、下划线，但是不推荐使用中文 虽然在本地开发没有什么问题，但是不能保证不会在其他的操作系统出现问题，我们的代码最终会部署到服务器上运行，我们的代码并不是一直跑在本地 (毕竟操作系统这些都是老外的东西)

# 文件后缀

  • 使用小写字线命名文件，不要出现中文字符
  • 扩展名标准是.html，当然也可以使用.htm
  • 多个单词可以使用- 或 _ 连接，建议使用- 字符如hello-vue.html

htm 是来源于老的 8.3 文件格式，DOS 操作系统只能支持长度为三位的后缀名，所以是 htm,但 在windows 下无所谓 HTM 与 HTML

www.baidu.com 域名(服务器地址)

# 访问路径

访问路径意思就是我们访问资源的路径

访问路径分为相对路径./和绝对路径D:\xxx\xxx

绝对路径表示从磁盘根路径开始的路径

# 相对路径

相对于当前文件夹

./index.html 当前路径下的index文件
../index.html 上级目录下的index文件
/index.html  这个表示域名下的index文件

# 代码注释

有时候我们会想给代码加一个说明 着重声明此段代码的作用 我们会使用注释 注释不会被网页解析出来

<!--我就是注释-->
<h1>hello</h1>
<!--这个是跳转链接-->
<a href="https://www.baidu.com"></a>

3
4

刚开始学习加注释是一个好习惯 但是没必要每一行都加 我们会在最重要的 不容易理解的代码加注释(其实主要是为了自己以后改代码的时候不至于看不懂......逃)
